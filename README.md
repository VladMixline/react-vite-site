# react-vite-site

Стартовый сайт на `React` и `Vite` с простым автосохранением в `GitHub`.

## Команды

- `npm run dev` — запускает сайт, Telegram API и watcher автосохранения.
- `npm run dev:telegram` — запускает сайт и Telegram API (без автокоммита). **Используйте для проверки формы.**
- `npm run dev:site` — запускает только Vite (форма без бота откроет Telegram вручную).
- `npm run telegram-api` — запускает только локальный API для автоматической отправки формы.
- `npm run telegram:chat-id` — помогает получить `TELEGRAM_CHAT_ID` после того, как вы напишете боту в Telegram.
- `npm run autosave` — запускает только watcher автосохранения.
- `npm run build` — собирает production-версию сайта.
- `npm run lint` — проверяет код через ESLint.
- `npm run ci` — запускает линт и production-сборку подряд.

По умолчанию локальный сайт открывается по адресу `http://localhost:5173/`.

## Как работает автосохранение

Когда запущен `npm run dev`, watcher отслеживает изменения в проекте, кроме `.git`, `node_modules` и `dist`.
После короткой паузы он:

1. добавляет изменения в git;
2. создаёт коммит вида `chore: auto save ...`;
3. отправляет изменения в GitHub.

Автор автокоммита берётся из локального `git user.name` / `git user.email`, а если они не заданы, используется автор последнего коммита в репозитории.

## GitHub

В репозитории настроен workflow `CI` через GitHub Actions. При каждом `push` и `pull request` он:

1. ставит зависимости через `npm ci`;
2. запускает `npm run ci`;
3. проверяет, что линт и production-сборка проходят без ошибок.

Для локального автопуша важно, чтобы у git уже был доступ к GitHub по сохранённым учётным данным или токену. Если доступ не настроен, автосинк всё равно создаст коммит, но сообщит в консоли, что `push` не выполнился.

## Telegram-форма

Для полностью автоматической отправки формы в Telegram:

### Локальная разработка

1. Создайте Telegram-бота через [@BotFather](https://t.me/BotFather): отправьте `/newbot`, придумайте имя.
2. Скопируйте `.env.example` в `.env.local`.
3. Вставьте токен бота в `TELEGRAM_BOT_TOKEN`.
4. Напишите боту в Telegram любое сообщение (например, `/start`).
5. Выполните `npm run telegram:chat-id` и скопируйте нужный `TELEGRAM_CHAT_ID`.
6. Сохраните `TELEGRAM_CHAT_ID` в `.env.local`.

Запустите `npm run dev:telegram` — форма будет отправляться автоматически в бота.

### Публикация (Vercel)

1. Создайте проект на [Vercel](https://vercel.com) и подключите репозиторий.
2. В настройках проекта → Environment Variables добавьте:
   - `TELEGRAM_BOT_TOKEN` — токен от BotFather
   - `TELEGRAM_CHAT_ID` — ID чата после `npm run telegram:chat-id`
3. Задеплойте проект.

Форма на опубликованном сайте будет отправлять ответы в Telegram автоматически.

Если бот не настроен, сайт откроет Telegram с уже подготовленным сообщением для ручной отправки.

## Следующие шаги
